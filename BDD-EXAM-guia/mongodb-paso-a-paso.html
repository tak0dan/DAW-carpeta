02. Query builder

Como construir consultas complejas desde consultas simples

Esta pagina responde justo a la parte mas importante del encargo: no solo da consultas hechas, tambien enseña como crecer una consulta con uno, dos, tres o cinco filtros y como escribir filtros dependientes.

Inicio Fundamentos Updates y arrays Biblioteca de recetas

Metodo general

  1. Escribe la frase del ejercicio con lenguaje claro.
  2. Detecta la coleccion.
  3. Saca el primer filtro obligatorio.
  4. Anade filtros independientes como campos hermanos.
  5. Si una parte dice "o bien", crea un $or.
  6. Si una parte depende de otra rama, mete varios filtros dentro de esa rama del $or.
  7. Si al final quieres pocos campos, aplica proyeccion.

Ejemplo 1: 1 filtro

Enunciado

Quiero las pizzas.

Consulta

db.comidas.find({
  categoria: "Pizza"
})

Ejemplo 2: 2 filtros

Enunciado

Quiero las pizzas veganas.

Consulta

db.comidas.find({
  categoria: "Pizza",
  vegano: true
})

Como las dos condiciones son a la vez, se ponen en el mismo objeto.

Ejemplo 3: 3 filtros

Enunciado

Quiero las pizzas veganas por debajo de 20.

Consulta

db.comidas.find({
  categoria: "Pizza",
  vegano: true,
  precio: {$lt: 20}
})

Ejemplo 4: 5 filtros

Enunciado

Quiero pizzas veganas, por debajo de 20, con valoracion mayor que 4.5 y dadas de alta este ano.

Consulta

db.comidas.find({
  categoria: "Pizza",
  vegano: true,
  precio: {$lt: 20},
  valoracion: {$gt: 4.5},
  fecha_alta: {
    $gte: ISODate("2026-01-01"),
    $lte: ISODate("2026-12-31")
  }
})

Filtros dependientes: patron clave

Si el enunciado dice algo como:

Si el item es A, entonces A debe cumplir B. O si el item es C, entonces C debe cumplir D. Y todo ademas debe estar entre X e Y.

Tu estructura mental debe ser esta:

db.comidas.find({
  $and: [
    {
      $or: [
        {categoria: "Pizza", vegano: true},
        {categoria: "Postre", disponible: true}
      ]
    },
    {
      fecha_alta: {
        $gte: ISODate("2026-01-01"),
        $lte: ISODate("2026-05-31")
      }
    }
  ]
})

Cada rama del $or lleva sus propias condiciones internas. Luego las reglas comunes van fuera, en el $and.

Construccion paso a paso de una condicion dependiente

Paso 1

db.comidas.find({
  categoria: "Pizza"
})

Paso 2

db.comidas.find({
  categoria: "Pizza",
  vegano: true
})

Paso 3

db.comidas.find({
  $or: [
    {categoria: "Pizza", vegano: true},
    {categoria: "Postre", disponible: true}
  ]
})

Paso 4

db.comidas.find({
  $and: [
    {
      $or: [
        {categoria: "Pizza", vegano: true},
        {categoria: "Postre", disponible: true}
      ]
    },
    {
      precio: {$lt: 20}
    }
  ]
})

Paso 5

db.comidas.find({
  $and: [
    {
      $or: [
        {categoria: "Pizza", vegano: true},
        {categoria: "Postre", disponible: true}
      ]
    },
    {
      precio: {$lt: 20}
    },
    {
      fecha_alta: {
        $gte: ISODate("2026-01-01"),
        $lte: ISODate("2026-05-31")
      }
    }
  ]
})

Lectura

(Pizza vegana OR postre disponible) AND precio bajo AND fecha dentro del rango.

Patrones listos para enunciados frecuentes

Frase del enunciado Patron
A y B y C {a: ..., b: ..., c: ...}
A o B {$or: [{a: ...}, {b: ...}]}
A y (B o C) {$and: [{a: ...}, {$or: [{b: ...}, {c: ...}]}]}
Si es A, entonces B; si es C, entonces D {$or: [{tipo: "A", b: ...}, {tipo: "C", d: ...}]}
... y ademas entre fechas Anade fecha: {$gte: ..., $lte: ...} fuera o dentro de la rama segun corresponda.

Nested fields y arrays dentro de consultas complejas

Campo interno

db.comidas.find({
  "origen.pais": "Italia",
  precio: {$lt: 20}
})

Array simple

db.comidas.find({
  ingredientes: {$all: ["tomate", "albahaca"]}
})

Array de objetos

db.comidas.find({
  stock: {
    $elemMatch: {
      sede: "Valencia",
      cantidad: {$lt: 10}
    }
  }
})

12 consultas modelo explicadas por dificultad

01. db.comidas.find({categoria: "Pizza"})
02. db.comidas.find({categoria: "Pizza", vegano: true})
03. db.comidas.find({categoria: "Pizza", vegano: true, precio: {$lt: 20}})
04. db.comidas.find({categoria: "Pizza", vegano: true, precio: {$lt: 20}, valoracion: {$gt: 4.5}})
05. db.comidas.find({categoria: "Pizza", vegano: true, precio: {$lt: 20}, valoracion: {$gt: 4.5}, fecha_alta: {$gte: ISODate("2026-01-01"), $lte: ISODate("2026-05-31")}})
06. db.comidas.find({$or: [{categoria: "Pizza"}, {categoria: "Postre"}]})
07. db.comidas.find({$and: [{$or: [{categoria: "Pizza"}, {categoria: "Postre"}]}, {valoracion: {$gt: 4}}]})
08. db.comidas.find({$or: [{categoria: "Pizza", vegano: true}, {categoria: "Postre", disponible: true}]})
09. db.comidas.find({$and: [{$or: [{categoria: "Pizza", vegano: true}, {categoria: "Postre", disponible: true}]}, {fecha_alta: {$gte: ISODate("2026-01-01"), $lte: ISODate("2026-05-31")}}]})
10. db.comidas.find({$and: [{$or: [{categoria: "Pizza", vegano: true}, {categoria: "Postre", disponible: true}]}, {fecha_alta: {$gte: ISODate("2026-01-01"), $lte: ISODate("2026-05-31")}}, {ingredientes: {$nin: ["ajo"]}}]})
11. db.comidas.find({$and: [{$or: [{"origen.pais": "Italia", categoria: "Pizza"}, {"origen.pais": "Espana", categoria: "Principal"}]}, {precio: {$lt: 20}}]})
12. db.comidas.find({$and: [{$or: [{categoria: "Pizza", vegano: true}, {categoria: "Postre", disponible: true}]}, {fecha_alta: {$gte: ISODate("2026-01-01"), $lte: ISODate("2026-05-31")}}, {stock: {$elemMatch: {sede: "Valencia", cantidad: {$lt: 10}}}}]})

Antes de pasar a aggregation

  • Sabes cuando basta con AND implicito y cuando necesitas $and.
  • Sabes meter varias condiciones dentro de una rama del $or.
  • Sabes colocar una condicion comun fuera de las ramas.
  • Sabes anadir fechas, arrays y nested fields a una consulta compleja.

Cuando domines estos patrones, la pagina de recipes te servira mucho mejor porque ya sabras desmontar y reconstruir cada consulta.
