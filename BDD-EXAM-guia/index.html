Como usar esta guia

Ruta de aprendizaje

  1. Empieza por Fundamentos si todavia no dominas shell, colecciones y find().
  2. Pasa a Construir consultas para ver como una consulta simple crece hasta tener 5 filtros, ramas OR y fechas.
  3. Despues entra en Updates y arrays para aprender a anadir, quitar, cambiar y recalcular campos.
  4. Termina con Aggregation y avanzado para pipelines, $lookup, indices y explain().
  5. Usa Biblioteca de recetas como chuleta de examen o de practica.

Regla principal

No intentes escribir la consulta final de golpe. En MongoDB funciona mejor este orden:

  1. Haz que devuelva algo.
  2. Anade un filtro.
  3. Anade otro filtro.
  4. Agrupa ramas con $or o $and si hace falta.
  5. Solo al final anade proyeccion, orden, limite o pipeline.

Ideas fuerza para el examen

1. AND implicito

Si escribes varios campos dentro del mismo objeto, MongoDB los une con AND automaticamente.

2. OR necesita arrays

Cuando una condicion puede seguir dos caminos, usa $or: [ {...}, {...} ].

3. Primero match, luego detalle

Construye el filtro principal antes de pensar en proyeccion, sort o pipeline.
