De cero a 3 filtros

Paso 1: leer todo

db.comidas.find()

Paso 2: 1 filtro

db.comidas.find({
  categoria: "Pizza"
})

Paso 3: 2 filtros

db.comidas.find({
  categoria: "Pizza",
  vegano: true
})

Paso 4: 3 filtros

db.comidas.find({
  categoria: "Pizza",
  vegano: true,
  precio: {$lt: 20}
})

Lectura mental

Pizzas AND veganas AND con precio menor que 20.

Idea clave

Campos hermanos dentro del mismo objeto = AND implicito.

Comparadores basicos

Mayor que

db.comidas.find({
  precio: {$gt: 10}
})

Menor o igual

db.comidas.find({
  precio: {$lte: 10}
})

Entre dos valores

db.comidas.find({
  precio: {$gte: 5, $lte: 15}
})

No igual

db.comidas.find({
  categoria: {$ne: "Postre"}
})

Uno de varios

db.comidas.find({
  categoria: {$in: ["Pizza", "Postre"]}
})

Ninguno de varios

db.comidas.find({
  categoria: {$nin: ["Postre", "Principal"]}
})

Proyeccion: devolver solo algunos campos

Cuando dices "quiero ver solo nombre y precio" no estas filtrando documentos: estas filtrando campos.

Solo nombre

db.comidas.find(
  {},
  {nombre: 1}
)

Nombre y precio

db.comidas.find(
  {},
  {nombre: 1, precio: 1, _id: 0}
)

Ocultar un campo

db.comidas.find(
  {},
  {ingredientes: 0}
)

Orden y paginacion

Orden ascendente

db.comidas.find().sort({
  precio: 1
})

Orden descendente

db.comidas.find().sort({
  precio: -1
})

Varias claves

db.comidas.find().sort({
  categoria: 1,
  precio: -1
})

Primeros 5

db.comidas.find().limit(5)

Pagina 3 de 10

db.comidas.find()
.sort({nombre: 1})
.skip(20)
.limit(10)

Consejo

Normalmente primero ordenas y despues paginas.

Arrays, regex y fechas en nivel basico

Array contiene valor

db.comidas.find({
  ingredientes: "arroz"
})

Empieza por

db.comidas.find({
  nombre: /^Pizza/
})

Rango de fechas

db.comidas.find({
  fecha_alta: {
    $gte: ISODate("2026-01-01"),
    $lte: ISODate("2026-12-31")
  }
})

Checklist minimo antes de pasar al siguiente nivel

  • Sabes la diferencia entre filtrar documentos y proyectar campos.
  • Sabes usar $gt, $lt, $in y $ne.
  • Sabes ordenar y limitar resultados.
  • Sabes leer una consulta con 2 o 3 filtros seguidos.

Cuando esto te salga natural, pasa a la pagina de construccion de consultas complejas.
